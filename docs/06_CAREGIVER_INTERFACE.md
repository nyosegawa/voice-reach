# 06 — 介護者側インターフェースとエコシステム

## 1. 設計方針

患者側の体験に集中しがちだが、コミュニケーションは双方向である。介護者側の体験設計が不十分だと、システム全体の価値が半減する。

### 1.1 介護者が必要とする情報

- 患者が今何を言おうとしているか（入力途中の見える化）
- 患者の現在の感情・体調の推定値（リアルタイム表示）
- VLMが認識している環境情報（部屋の状況、来訪者等）
- 過去の発話ログ
- 緊急アラート
- 「今話しかけていいか」の状態表示
- 引き継ぎ情報（前の介護者とのやりとり）

### 1.2 介護者の利用シーン

| シーン | デバイス | 必要機能 |
|---|---|---|
| ベッドサイドで対面 | 不要（直接聴く） | 患者の音声合成出力 |
| 別室にいる | スマートフォン | アラート通知、発話ログ |
| 外出中 | スマートウォッチ | 緊急通知のみ |
| 引き継ぎ時 | タブレット/PC | 会話ログ、状態サマリ |
| 医師との面談 | PC | レポート出力 |

### 2.4 YesNo質問モード

患者がフラストレーションを感じている場合や、候補が当たらない場合に、介護者側から絞り込む。

```
介護者の操作:
  1. [YesNo質問] をタップ
  2. 質問カテゴリを選択: 体のこと / 気持ちのこと / 環境のこと / その他
  3. 患者側の画面に [はい] [いいえ] の大きなボタンが表示
  4. 患者はチラ見 + タップで回答

介護者の画面:
  Q: "体のどこかが痛いですか？" → [はい]
  Q: "上半身ですか？" → [はい]
  Q: "背中ですか？" → [はい]
  → "背中が痛い" を確定
```

## 3. 通知設計

### 3.1 通知レベル

| レベル | 条件 | 通知方法 |
|---|---|---|
| 緊急 | 患者がトリプルタップ / バイタル異常 | 全デバイス: 音+バイブ+画面点灯 |
| 高 | 患者が「人を呼んで」を選択 | スマホ: バイブ+通知 |
| 中 | 患者が発話（別室にいる場合） | スマホ: 通知のみ |
| 低 | パッシブ検出（疲労、不快感） | スマホ: サイレント通知 |
| 情報 | システム状態（バッテリー低下等） | アプリ内のみ |

### 3.2 スマートウォッチ通知

```
緊急時:
  ┌──────────────┐
  │ ⚠️ 緊急       │
  │ 太郎さんが     │
  │ 助けを求めて   │
  │ います         │
  │ [確認]        │
  └──────────────┘

通常時:
  ┌──────────────┐
  │ 💬 太郎さん    │
  │ "水が欲しい"   │
  │ [OK] [後で]   │
  └──────────────┘
```

## 4. 複数介護者間の引き継ぎ

### 4.1 引き継ぎサマリの自動生成

介護者の交代時に、前のシフト中の要約を自動生成する。

```
[引き継ぎサマリ] 2026/02/17 午前シフト（花子）

■ コミュニケーション概要
  - 朝7時に起床。「おはよう」から始まり、
    体調は「まあまあ」とのこと。
  - 背中のかゆみを訴え（07:16）。
    塗り薬を塗布して改善。
  - 10時にリハビリのSTが来訪。
    約30分の視線入力訓練を実施。

■ 要注意事項
  - 背中の不快感が断続的に続いている模様
    （感情データで午前中にネガティブ偏位が複数回）
  - 午前中のまばたき頻度がやや高め。疲労に注意。

■ 投薬記録
  - 8:00 リルゾール 服用済み
  - 次回: 15:00 エダラボン

■ システム状態
  - 10:00にキャリブレーション実施（良好）
  - バッテリー: 72%
```

### 4.2 共有ダッシュボード

複数の介護者（家族、看護師、ヘルパー）がアクセスできるWebダッシュボード。

```
権限レベル:
  - 管理者（家族）: 全機能アクセス、PVP編集、設定変更、プライバシー設定管理
  - 介護者: 会話ログ閲覧、状態確認、YesNo質問
  - 医師: 状態レポート閲覧、バイタル履歴
  - ゲスト: 対面時の会話のみ（ログ閲覧不可）
```

## 5. 医療連携

### 5.1 医師向けレポート

定期通院時や遠隔診療時に提出するレポートを自動生成する。

```
[医師向けレポート] 2026/02/01〜02/17

■ コミュニケーション能力
  - 1日あたりの平均発話数: 25回（前月比 -3回）
  - 平均意図到達時間: 3.8秒（前月比 +0.4秒）
  - 自由入力の使用頻度: 1日2回（前月とほぼ同等）
  - 視線追跡の精度: 安定（大きな低下なし）

■ 入力チャネルの変化
  - 指センサー感度: 感度70→75に上方調整（2/10）
  - 顔面表情信号: 口角変化の信頼度が0.3→0.2に低下
  - まばたき・瞳孔信号: 安定

■ 感情傾向の変化
  - 午前はポジティブ寄り、午後に低下する傾向（継続）
  - 背中の不快感に関する発話が増加（前月比 +40%）

■ 特記事項
  - 2/10に指センサー感度を調整。操作性改善。
  - 2/14に新しい介護者（ヘルパーB）を顔登録。
```

### 5.2 看護記録との連携

既存の看護記録システム（電子カルテ）へのデータエクスポート機能。

```
エクスポート形式:
  - CSV: 発話ログ、バイタル連携データ
  - PDF: 状態サマリレポート
  - HL7 FHIR: 医療情報標準規格（将来対応）
```

## 6. プライバシー設定UI

### 6.1 カメラデータの送信制御

VLM環境認識に使用する外向きカメラのデータ送信について、患者・家族が細かく制御できるUIを提供する（レポートB5のプライバシー要件に準拠）。

```
┌────────────────────────────┐
│ プライバシー設定             │
├────────────────────────────┤
│                            │
│ ■ カメラデータ送信制御       │
│                            │
│ 外向きカメラ映像            │
│   ローカル処理のみ  [●━━━] │
│   クラウドVLM送信   [━━━○] │
│                            │
│ クラウド送信時のオプション   │
│   [✓] 送信前に毎回確認する  │
│   [✓] 人物を匿名化して送信  │
│   [ ] 常時送信を許可        │
│                            │
│ ■ クラウド送信先            │
│   [✓] Gemini 2.5 Flash     │
│       (Google ゼロリテンション) │
│   [ ] GPT-4o (OpenAI)      │
│                            │
│ ■ データ保持設定            │
│   ローカル映像保持: [5分  ▼]│
│   分析後の即時削除: [✓]     │
│   ログ保持期間:     [30日 ▼]│
│                            │
│ ■ 内向きカメラ（患者の顔）  │
│   クラウド送信: 常時禁止     │
│   (変更不可 - 安全設計)     │
│                            │
│ [変更を保存]  [初期設定に戻す]│
└────────────────────────────┘
```

設計原則:
- **デフォルトはローカル処理のみ**。クラウド送信はオプトイン（明示的な有効化が必要）
- 内向きカメラ（患者の顔・目）の映像は**一切クラウドに送信しない**（設計レベルで禁止。設計書01, 09参照）
- クラウドVLM送信時は、VLMプロバイダのゼロリテンションポリシーを確認済みのサービスのみ選択可能
- 送信制御の変更には管理者（家族）権限が必要
- 全データ送信イベントは監査ログに記録

### 6.2 権限別のカメラアクセス

```
権限とカメラデータの関係:
  - 管理者（家族）: プライバシー設定の変更権限あり。VLM環境情報閲覧可
  - 介護者: VLM環境情報（テキスト記述のみ）閲覧可。カメラ映像直接閲覧不可
  - 医師: VLM環境情報閲覧不可（状態レポートに含まれる範囲のみ）
  - ゲスト: 一切アクセス不可
```

## 7. 日本の福祉制度情報

### 7.1 補装具費支給制度

VoiceReachの導入に際し、日本の公的補助制度を最大限活用することが重要である（レポートA2 Section 4.2）。

**補装具費支給制度（厚生労働省）:**

| 項目 | 内容 |
|---|---|
| 分類 | 重度障害者用意思伝達装置 |
| 購入基準額 | **¥480,600**（環境制御機能・通信機能付き） |
| 自己負担上限 | ¥0（生活保護・非課税世帯）/ ¥37,200（課税世帯） |
| 耐用年数 | 5年 |
| 再支給 | ALS等の進行性疾患では病状変化に応じて再支給可能 |
| 対象者 | 身体障害者手帳（肢体不自由1-2級 かつ 言語障害3級）または指定難病 |
| 視線入力装置加算 | Tobii PCEye 5等 ¥242,000（オプション装着として給付対象） |

**日常生活用具給付等事業（市区町村）:**

| 項目 | 内容 |
|---|---|
| 管轄 | 市区町村 |
| 分類 | 情報・意思疎通支援用具 |
| 利点 | 申請手続きが簡易（正式なアセスメント不要） |
| 制約 | 進行性疾患での再支給なし（一回限り） |

**VoiceReachへの実務的示唆:**
- 課税世帯のALS患者でも自己負担は最大¥37,200で意思伝達装置を取得可能
- VoiceReachが「重度障害者用意思伝達装置」としての補装具認定を取得できれば、¥480,600基準額の範囲内で機器一式（Mac mini + カメラ + 指入力デバイス）を給付対象にできる可能性がある
- 認定取得にはテクノエイド協会（RESJA）との連携、および臨床評価データの蓄積が必要
- VoiceReachの指入力デバイス（BOM $41 = 約¥6,200）は、圧電素子式スイッチの補装具基準額¥53,400の範囲内に収まる（レポートA6）
